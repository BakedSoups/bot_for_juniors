## Aim of this project

### Personal reasoning 
I found that ai chat bots are great for getting full solutions/awnsers.

but  I personally find that as a junior, I learn best by trying to solve a problem myself. 
maybe dowwn the line, when I'm a senior who truly understands the ins and outs of programming, I can hand off task to AI, but for now, I want to be able to solve problems on my own. 

So as I start my coding journey, I could go all the way with no ai, but there are still some small caveats in documents and such that I would like to learn.
Additionally, there are times where I want to ask a question like "should I abstract to springboot?" or "is this architecture good?" which can be great for ai to awnser. 


#### TLDR 
I see the potential for ai for helping me learn, but I also want to make sure that I am learning the fundamentals and not just relying on ai for everything.


## How this app works 
it utilizes a offline ai model, which is instructed to never give me the full solutuion unless I ask for it. I also will have it so if I fall into the habit of copying an entire file and simply asking for an explation, it will refuse to do so and instead ask me to break it down into smaller pieces. 

addtionally this will be a oneshot app, meaning I need to start a new convo with the correct context

### Future plans 
- documentation parser, which will allow me to ask questions about documentation and get awnsers without having to read through the entire thing. 
- obsidian plugin, which will allow me to ask questions about my notes and get awnsers without having to read through the entire thing.
